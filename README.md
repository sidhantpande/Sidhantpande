## 🎯 What I've Built

### 🚀 [Study Buddy](https://github.com/Sidhant185/StuddyBuddy) — AI-Powered Learning Platform  
*Hackathon Winner · Live Campus Product*

A complete learning ecosystem that evaluates code **and learns from it**. Monaco Editor for real-time coding, instant execution, AI-powered feedback, and an adaptive practice engine.

**Why this matters:**
- Built in **24 hours** at a hackathon, now used by 100+ students
- Real architecture: Firebase Auth with SSO, Firestore real-time updates, CI/CD ready
- Complex AI pipeline: code execution → evaluation → weakness identification

**Tech Stack:**
```
Frontend:    React 18, Vite, Tailwind CSS, Monaco Editor
Backend:     Node.js, REST APIs, Piston API integration
Database:    Firebase Firestore, Firebase Auth
AI/ML:       Gemini 2.0 API, context-aware evaluation
DevOps:      Environment-based configs, error handling
```

**Features:**
- Monaco Editor (VS Code in the browser)
- Piston API for sandboxed code execution
- Gemini AI evaluates code + explains weaknesses
- Smart practice engine: weighted algorithm learns your weak areas
- Admin dashboard for bulk contest management
- Domain-restricted SSO (@vedamsot.org)

### 🏢 [Vedam Open Source — Admin Dashboard](https://github.com/vstopensource/dashboard)  
*Enterprise-Grade Community Management*

Professional dashboard for tracking 500+ students, their GitHub contributions, form submissions, and real-time analytics.

**Why this matters:**
- Enterprise architecture patterns in a student project
- Handles **500+ member lists** with virtual scrolling (zero lag)
- Real GitHub API integration with rate limiting & smart caching
- **Security scanning** on every deployment (CI/CD via GitHub Actions)
- Full accessibility: ARIA labels, keyboard navigation, high contrast mode

**Tech Stack:**
```
Frontend:    Vanilla JS (ES6+), Custom CSS, Accessible Design
Backend:     Firebase Firestore (real-time DB)
APIs:        GitHub REST API, GitHub GraphQL
DevOps:      Netlify CI/CD, GitHub Actions, Security scanning
Architecture: Virtual scrolling, request deduplication, exponential backoff
```

**Key Challenges Solved:**
- Caching GitHub API responses (avoid rate limits)
- Virtual scrolling for large datasets
- Real-time member list updates
- Automated security scanning before deployment
